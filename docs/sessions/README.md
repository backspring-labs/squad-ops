# Development Sessions

This folder captures detailed development session notes, progress logs, and implementation summaries.

## Purpose

Sessions are different from other documentation:
- **SIPs** (`docs/SIPs/`) - Formal proposals and specifications
- **Retros** (`docs/retro/`) - Post-mortem analysis and lessons learned
- **Sessions** (`docs/sessions/`) - Real-time development progress and session summaries

## Naming Convention

Format: `YYYY-MM-DD-topic-brief-description.md`

Examples:
- `2025-10-12-sip027-phase1-implementation.md`
- `2025-10-15-warmboot-run056-session.md`
- `2025-10-20-prefect-integration-exploration.md`

## What to Include

### Implementation Sessions
- What was built
- Code changes made
- Test coverage
- Technical decisions
- Blockers encountered
- Next steps

### WarmBoot Run Sessions
- Run configuration
- Real-time observations
- Agent interactions
- Issues encountered
- Wrap-up analysis

### Exploration/Spike Sessions
- What was explored
- Findings and insights
- Prototypes/experiments
- Recommendations
- Decision made

### Debug/Troubleshooting Sessions
- Problem description
- Investigation steps
- Root cause analysis
- Fix implemented
- Prevention measures
